                                                                                                              # AQS的作用

                                                                                                              AQS的全称是AbstractQueuedSynchronizer类，大部分的JUC并发工具类都是通过内部实现一个Sync类，继承AQS类来实现AQS的功能，以并发的控制线程和资源。通过实现AQS，可以实现以下的功能：

                                                                                                              • 阻塞队列等待
                                                                                                              • 共享/独占
                                                                                                              • 公平/非公平
                                                                                                              • 可重入
                                                                                                              • 允许中断

                                                                                                              # 内部原理

                                                                                                              # 核心组成

                                                                                                              • state状态：
                                                                                                                • 会被并发修改到，因此volatile修饰保证并发安全，并且其修改方法使用unsafe保证原子性。
                                                                                                                • 其含义根据实现类的不同而不同，如Semaphore里，表示“许可证的数量”；在CountDownLatch中表示“还需要倒数的计数数量”；ReentrantLock中表示锁的占有情况，也就是重入次数。
                                                                                                                • getState()、setState()、compareAndSetState()
                                                                                                              • 控制线程抢锁的FIFO队列（同步等待队列）：
                                                                                                                • 用于存放等待的线程，AQS的排队管理器
                                                                                                                • 队列是双向链表，所有的线程都放在队列里
                                                                                                                • 队列分为同步等待队列（上述）和条件等待队列（Condition）
                                                                                                              • 协作工具类需要去实现的方法：
                                                                                                                • isHeldExclusion()：该线程是否正在独占资源，只有用到Condition才需要去实现。
                                                                                                                • tryAcquire(int)：独占方式，尝试获取资源，成功返回true，失败返回false
                                                                                                                • tryRelease(int)：独占方式，尝试释放资源，成功返回true，失败返回false
                                                                                                                • tryAcquireShared(int)：共享方式，尝试获取资源，负数表示失败，0表示成功，但没有可用剩余资源；正数表示成功，并且有剩余资源
                                                                                                                • tryReleaseShared(int)：共享方式，尝试释放资源，如果释放后允许唤醒后续等待节点返回true，否则返回false。
                                                                                                              • 资源共享方式：
                                                                                                                • Exclusion：独占，只能有一个线程执行，如ReentrantLock
                                                                                                                • Share：共享，多个线程可以共同执行，如CountDownLatch、Semaphore

                                                                                                              # 同步等待队列原理

                                                                                                              AQS的同步等待队列是一种双向链表结构的FIFO队列（先入先出），队列以Node为节点，节点中除了包含前驱指针和后区指针，还包含了线程实例和信号状态。

                                                                                                              # 条件等待队列原理

                                                                                                              Condition是一个工具类，只有线程具备某些条件的时候，这些等待条件才会被唤醒，是一个单链表结构，也是由Node节点组成。

                                                                                                              # 实现方式

                                                                                                              1. 定义一个内部类Sync，继承AQS
                                                                                                              2. 将需要实现的同步器的逻辑都映射到Sync对应的方法
