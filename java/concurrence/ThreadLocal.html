                                                                                                                            # 使用场景及特点

                                                                                                                            # 场景

                                                                                                                            • 线程内部独享，不同线程之间不共享
                                                                                                                              • 工具类实例，线程共享的话不安全，如SimpleDateFormat和Random类
                                                                                                                            • 每个线程内有独享的全局变量
                                                                                                                              • 如拦截器获取用户信息，可以在线程内部贯穿使用，避免参数传递的麻烦
                                                                                                                            • Spring中的应用
                                                                                                                              • RequestContentHolder：每个请求都是一个线程，线程中用于保存请求信息
                                                                                                                              • DatetimeContentHolder：保存时间

                                                                                                                            # 特点

                                                                                                                            • 线程内部独有，线程之间不共享，因此也是线程安全的。
                                                                                                                            • 要避免导致内存泄漏，使用拦截器要在方法结束后显示的调用remove方法。
                                                                                                                            • 使用initialValue()或者set()避免空指针。
                                                                                                                            • 如果ThreadLocal中set的对象本身就是线程之间的共享对象，那么即便是用ThreadLocal取出来，线程之间依旧是共享的，比如static对象。

                                                                                                                            # 作为工具类实现

                                                                                                                            public class ThreadExclusiveTool {
    public static ExecutorService threadPool = Executors.newFixedThreadPool(10);
    
    public static void main(String[] args) {
        for (int i = 0; i < 1000; i++) {
            int finalI = i;
            threadPool.submit(() ->{
                String dateParse = new ThreadExclusiveTool().dateParse(finalI);
                System.out.println(dateParse);
            });
        }
        threadPool.shutdown();
    }
    public String dateParse(int seconds) {
        Date date = new Date(1000 * seconds);
        // 4. 使用时通过调用 get() 方法获得
        SimpleDateFormat format = ThreadSafeFormat.dateFormatThreadLocal.get();
        return format.format(date);
    }
}
class ThreadSafeFormat{
    // 1. 声明静态的ThreadLocal
    public static ThreadLocal<SimpleDateFormat> dateFormatThreadLocal = new ThreadLocal() {
        // 2. 重写 initialValue() 方法进行初始化，创建需要的对象
        @Override
        protected Object initialValue() {
            // 3. 初始化时返回一个新的需要的对象
            return new SimpleDateFormat("yyyy-MM-dd hh:mm:ss");
        }
    };
}

                                                                                                                            # API

                                                                                                                            • initialValue()：返回ThreadLocal的初始值（或者叫做通过这个方法设置ThreadLocal的初始值或者默认值），这个方法是延迟加载的，调用get()的时候才会返回，在get()方法中如果get到的是空，那么会调用initialValue方法来进行初始化。这个方法没有默认实现，可以允许用户自己实现，默认返回null。
                                                                                                                            • get()：获取ThreadLocal保存的对象的值，如果已经set()了对象，取到对象；如果没有则调用initialValue进行初始化，如果没有设置初始化方法返回null。
                                                                                                                            • remove()：删除ThreadLocal的值，如果调用了这个方法，再次调用get()，也会触发initialValue()。

                                                                                                                            # 底层源码及原理

                                                                                                                            # 存储结构

                                                                                                                            如上图所示，每个线程（Thread）中都有一个ThreadLocalMap，使用keyValue的形式保存多个ThreadLocal。

                                                                                                                            即在Thread类中，有一个ThreadLocalMap成员变量来保存ThreadLocal。Map中保存的键值对中的key是ThreadLocal，value是需要保存的对象。

                                                                                                                            因此，ThreadLocal所保存的对象实际上是保存在了Thread对象中，而不是保存在ThreadLocal对象中，ThreadLocal只是个key。

                                                                                                                            # ThreadLocalMap

                                                                                                                            ThreadLocalMap是定义在Thread中的静态类，其中最核心的也就是保存对象使用的是一个Entry数组，如下图所示：

                                                                                                                            其原理可以近似的理解为HashMap，但是解决Hash冲突的方式不同，Entry采用的是线性探测法解决Hash冲突，而不是用链表拉链和红黑树。如下图所示，如果Hash冲突了，不会拉链后者红黑树，而是直接找下一个空位置填进去。

                                                                                                                            同时，构造函数中，key是使用弱引用进行赋值，也就是key是可以被垃圾回收器回收，但是value是强引用，无法被垃圾回收，因此value对象有可能造成OOM。在ThreadLocal的set()、remove()、rehash()方法中，会判断key是否为null，如果key是null，就会将其value也设置为null，可以一定程度上帮助GC。但是一个ThreadLocal不被使用后，也很难再调用这些方法。因此使用完成后一定显式的调用remove()方法。

                                                                                                                            # get()方法

                                                                                                                            public T get() {
    // 1. 获取当前线程
    Thread t = Thread.currentThread();
    // 2. 获取本线程的 ThreadLocalMap
    ThreadLocalMap map = getMap(t);
    if (map != null) {
        // 3. 获取TheadLocal
        ThreadLocalMap.Entry e = map.getEntry(this);
        if (e != null) {
            // 4. 获取对象
            @SuppressWarnings("unchecked")
            T result = (T)e.value;
            return result;
        }
    }
    // 5. 如果没有值则进行初始化
    return setInitialValue();
}

                                                                                                                            # set()方法

                                                                                                                            public void set(T value) {
    // 1. 获取当前线程的引用
    Thread t = Thread.currentThread();
    // 2. 获取当前线程的ThreadLocalMap
    ThreadLocalMap map = getMap(t);
    if (map != null)
        // 如果存在Map则set值
        map.set(this, value);
    else
        // 如果不存在Map则创建并且set值
        createMap(t, value);
}

                                                                                                                            # remove()

                                                                                                                            public void remove() {
    //1. 获取到当前线程的ThreadLocalMap
    ThreadLocalMap m = getMap(Thread.currentThread());
    if (m != null)
        // 2. 删除这个ThreadLocal对应的value
        m.remove(this);
}
