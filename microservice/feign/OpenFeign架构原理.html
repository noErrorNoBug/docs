                                                                    在微服务系统中，我们经常用到RPC调用，OpenFeign就是基于Feign项目的一套声明式、模板化的Http客户端。OpenFeign对Feign进行了增强，支持Spring MVC的注解，同时整合了Eureka、Nacos、Gateway、LoadBalancer等组件，使得Feign的使用更加便捷。

                                                                    # 使用场景

                                                                    # 整合

                                                                    引入依赖（2020.0.x版本后需要手动引入LoadBalancer）：

                                                                    <dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-loadbalancer</artifactId>
</dependency>

                                                                    # 编写接口

                                                                    接口支持SpringMVC风格注解，OpenFeign自己提供的注解和配置扩展在后续的专门扩展配置的文章中统一整理。

                                                                    @FeignClient(value = "demo-nacos-provider",path = "/nacos/provider")
public interface CallNacosProviderFeign {
    @PostMapping("/simple")
    Position simpleProvider(@RequestBody Identity identity);
}

                                                                    @ApiOperationSupport(order = 2)
@ApiOperation(value = "直接调用Provider接口")
@GetMapping(value = "/direct",produces = {"application/json;charset=UTF-8"})
public Mono<Position> directCall() {
    Identity identity = Identity.builder().identity("demo-openfeign").dateTime(LocalDateTime.now()).build();
    Position position = feign.simpleProvider(identity);
    return Mono.just(position);
}

                                                                    # 核心原理

                                                                    官方文档中的组件结构，是以实现功能为维度的，下面对于组件的功能说明，我们以源码为维度。一方面这样展开比较好理解，另一方面，我们使用时，如果需要进行扩展和定制，也是以源码为维度去进行延伸。

                                                                    OpenFeign的设计架构如上图所示，左侧是大体执行流程，中间是核心组件，右边是其支持的主要组件，也是其主要的扩展点。

                                                                    OpenFeign的主要执行过程主要分为如下几个步骤：

                                                                    1. 通过动态代理机制生成声明式函数接口的实现类。
                                                                    2. Contract组件根据指定的契约配置识别函数式接口的声明。
                                                                    3. MethodHandler基于声明进行Request(Response)与Request Bean(Response Bean)的转换。
                                                                    4. Encoder/Decoder 对参数进行序列化和反序列化。
                                                                    5. Interceptor 可以由用户配置和添加，用于拦截请求。
                                                                    6. Logger记录远程调用日志。
                                                                    7. 用户可以指定Feign Client完成底层Http请求的调用。
